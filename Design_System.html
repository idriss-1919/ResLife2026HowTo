Design principles

Minimal

Remove until it breaks. If an element doesn't serve the learner, it doesn't belong on the surface. Whitespace is not empty — it is structural.

Structured

Every surface has a clear hierarchy. Eyebrow, title, body, footer — the order never changes. Predictability builds trust.

Human

This system serves people in a room, not pixels on a screen. Every choice asks: does this help the person sitting in the third row?

Intentional

Nothing is decorative. Color is earned, not given. A border means containment. A dark surface means emphasis. Every element has a reason.

Layout system

Grid

12-column grid. 16px (s-2) gutter. 32px (s-4) page margins. Content areas use 2-column and 3-column subdivisions. Slides use percentage-based padding (10–12%) for aspect-ratio independence.

Accessibility

Accessibility is structural, not decorative. Every decision — color, type size, motion, layout — was made with accessibility as a constraint.

Focus, motion, color-blind safety

All interactive elements use :focus-visible with a 2px offset outline. The system respects prefers-reduced-motion — all animations collapse to 0ms. The system is monochrome by default; semantic colors are never the sole indicator of meaning. All video ships with captions enabled by default.

01

Principles

Typography does all the work.

Size establishes importance. Weight creates emphasis. Spacing creates structure. You don't need color to build hierarchy when your type scale is doing its job.

Color is earned, not given.

The default state of everything is black, white, and gray. Color only appears when something needs to be distinguished from something else — not to look nice.

One idea per surface.

Every screen, card, and slide communicates exactly one thing. If you need two ideas, you need two surfaces. Whitespace is not empty — it's structural.

Components are text in boxes.

A card is text with a border. A button is text with a background. Every component should be describable in one sentence. If you need a paragraph, it's too complex.

02

The moment system

Not everything should feel the same. The system defines three design modes — called moments — that control when color, contrast, and scale enter the experience. This is the single most important concept in the system. If your team learns nothing else, they need to learn this.

Design Modes — the core concept

Every surface, slide, poster, and screen in the ResLife system exists in one of three modes. The mode determines how much visual energy the surface carries. Getting this right is the difference between a system that feels intentional and one that feels random.

1

Base

Default learning. White/gray surface, ink text, zero color. 80% of all surfaces live here. Calm, legible, invisible.

2

Pillar

Contextual meaning. Pillar color floods background or accents an element. Tells the learner which domain they're in.

3

Emphasis

Critical attention. Dark surface, oversized type, full-bleed color. The system gets loud. Use sparingly — once per five screens max.

The rule: If you can't name which mode a surface is in, it's wrong. Every screen should have a clear answer: Base, Pillar, or Emphasis. Mixed modes create visual confusion. Underusing Pillar makes color feel random. Overusing Emphasis makes nothing feel important.

Motion principles

Motion creates hierarchy, feedback, and energy. Four rules govern all animation in the system.

Entry — fade + rise

Content enters by fading in while translating up 8–16px. Duration: 200–300ms using ease-enter. Stagger children by 50ms for lists and cards.

opacity: 0 → 1; translateY(16px → 0)

Hierarchy — staggered reveal

When multiple elements appear together, they don't arrive simultaneously. The headline lands first, then body text, then supporting elements. 50ms delay between each.

delay: 0ms, 50ms, 100ms, 150ms

Feedback — instant

Hover states, button presses, toggle switches. 0–100ms. The user should never wonder if they clicked. Use ease-fast (100ms).

--ease-fast: 100ms ease-out

Reading surfaces — no motion

Body text, tables, data, reference content. These surfaces are for reading. No entrance animation. No scroll-triggered effects. Motion here is distraction.

transition: none; animation: none;

06

The slide system

Eleven slide types. Four structural (Cover, Section Divider, Content, Closing) give the deck its skeleton. Seven secondary types (Quote, Data, Image+Text, Activity, Framework, Process, Reflection) turn information into experience.

Slide discipline — the non-negotiables

One message per slide

If you need two points, make two slides. Every slide should be describable in a single sentence. If you catch yourself saying "and also..." — split it.

No bullet points. Ever.

Bullets are a crutch that signal the content hasn't been designed. If you have a list, use a Process slide, a Framework slide, or separate each item onto its own Content slide. The audience reads ahead on bullet slides — they stop listening.

Title: max 8–12 words, max 2 lines

If your title wraps to a third line, it's not a title — it's a paragraph. Cut it. The title is a promise, not a summary.

Body: max 2 lines, max 30 words

The body supports the title — it doesn't repeat it or expand into an essay. Two lines. If you need more, you need another slide.

Cover4–6 words1 short subtitle12 words

One visual OR one idea — never both heavy

A slide can carry a strong visual (photo, diagram, chart) or a strong idea (headline + body). When both compete for attention, neither wins. If your slide has a photo and three lines of text, remove two lines.

The squint test

Squint at the slide from arm's length. You should be able to identify the hierarchy instantly: what's the headline? Where's the body? What's the action? If everything blurs into the same weight, the slide needs to be redesigned.

No bullet lists. If you must, max 3 items.

Lists of 4+ items are a sign you haven't designed the content. Break each item into its own slide, or use a Process/Sequence layout. If absolutely necessary (steps, instructions), cap at 3 items — and each item gets one line max.

Max 1 visual element per slide

One photo, or one chart, or one diagram, or one icon. Never two. The visual is the anchor — if there are two, the eye bounces and neither lands. If you need two visuals, you need two slides.

Color usage for meaning

Monochrome by default

All charts use one color: ink (#005587). Differentiate series by opacity, not by adding new colors. Three opacity levels are enough for most datasets.

Semantic color (exception only)

Color carries meaning only for status. Green = success/complete. Red = error/at-risk. Amber = warning/in-progress. These are the only three additional colors in the system.

Never use color as the sole differentiator. Every colored element must also have a label, pattern, or position that conveys meaning without color. This ensures accessibility for colorblind users (~8% of men, ~0.5% of women).

Anti-patterns

The system becomes teachable when people can see what not to do. These are the five most common ways staff break the system — and why they fail.

✕ DON'T — Too much text

Community Standards Overview

Community standards are the foundation of residential life at UCLA. They establish shared expectations for behavior, mutual respect, and accountability among all residents. These standards are not just rules — they represent our collective commitment to creating spaces where every resident can thrive academically, socially, and personally.

Key areas include: noise policies, guest policies, shared space usage, alcohol and substance policies, conflict resolution procedures, and emergency protocols. Each area has specific guidelines that residents must follow.

Additional considerations: roommate agreements, floor meetings, community events, and reporting procedures. Staff should be prepared to address violations promptly and fairly while maintaining resident relationships.

Three paragraphs on one slide. The learner reads none of them.

  /* SURFACES */
  --white:  #FFFFFF;
  --gray:   #F4F4F3;

  /* TEXT — two values, WCAG AA on every surface */

  /* DARK SURFACE TEXT */
  --dark-body: #F1F1F1;       /* on #005587 — 7.0:1 AAA  */
  --dark-sub:  #C4C4C4;       /* on #005587 — 4.5:1 AA   */

  /* BORDER */
  --line:   #E0E0E0;          /* the only border color */

  --font: 'Karbon', system-ui, sans-serif;

  --t-xs:   0.75rem;          /* 12px — labels */
  --t-sm:   0.875rem;         /* 14px — captions */
  --t-base: 1rem;             /* 16px — body */
